Loga is a **functional language standard** designed for developers. It eliminates complex English grammar (tenses, clauses, inflections) and replaces it with simple **SVO (Subject-Verb-Object) logic**, ensuring your GitHub READMEs and code comments are understood globally without translators.

---

## 🛠 Why Loga? (為什麼使用邏格？)

* **No "Down" Ambiguity**: Replaces vague words with precise states (e.g., `stop`, `wrong`).
* **Phonetic Smoothing**: Simplifies hard-to-read words (e.g., `Experience` -> `Eberense`).
* **Sinitic Logic**: Follows the natural thinking flow of Chinese speakers, which is also the most direct path for AI and non-native speakers.

---

## 📏 Core Rules (核心規範)

### 1. Simple SVO (主謂賓結構)
No inversions.
* *Standard*: The script was updated by me.
* **Loga**: `Me update script already.`

### 2. State Clarity (狀態明確)
Avoid multi-meaning words like "down" or "get".
* `stop` = Shutdown / Not working
* `wrong` = Error / Bug / Fault
* `run` = Executing / Working

### 3. Spelling Smoothing (拼寫簡化)
Shorten long words and smooth complex consonants.
* `Experience` -> **Eberense**
* `Storm` -> **Sidom**
* `Environment` -> **Enviro**

### 4. Tense & Interrogative Labels
Verbs always remain in their base form. Tense and mood are expressed via labels (tags).

* **Past**: Add `already` | **Future**: Add `will`
* **Questions**: No inversion. Keep the SVO order and place the question word at the end.
    * `U noi what?` (What do you know?)
    * `U run sript how?` (How do you run the script?)
    * `U like Loga ma?` (Do you like Loga?)
    * `Me will fix wong code.` (I will fix the wrong code.)
    * `Me already fix wong code.` (I have fixed the wrong code.)
